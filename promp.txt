=====================================================

Here is a single comprehensive prompt you can give to a coding agent:

Create a complete Android application in Java (NOT Kotlin) named "VideoChatApp".

Requirements:

1. General

* Use Android Studio project structure.
* Use Java for all application code.
* Target latest stable Android SDK.
* Use AndroidX libraries only.
* Use Material Design 3 components where applicable.
* The project must build successfully using GitHub Actions.
* Use the latest Gradle version supported by GitHub Actions at the time of generation.
* Include all required Gradle files:

  * settings.gradle
  * build.gradle (Project)
  * build.gradle (App)
  * gradle.properties
  * gradle-wrapper.properties
  * gradlew
  * gradlew.bat
  * wrapper JAR references
* Ensure the project can be built from command line using:
  ./gradlew assembleDebug

2. Main Functionality
   Build a basic video chat application.

Features:

* Home screen after launch.
* User can start a video call.
* User can join a video call using a room ID.
* Display local camera preview.
* Display remote participant video.
* Mute/unmute microphone.
* Enable/disable camera.
* End call button.
* Handle runtime permissions(Take permissions at app startup):

  * Camera
  * Microphone
  * Internet


Use WebRTC for video communication.

3. Navigation Drawer
   Implement a left-side navigation drawer (hamburger menu).

Menu items:

* Home
* Settings
* Help
* About

Requirements:

* Drawer should open from left side.
* Material Design navigation drawer.
* Home opens main screen.
* Settings opens SettingsActivity.
* Help opens HelpActivity.
* About opens AboutActivity.

4. Settings Screen
   Create SettingsActivity.

Requirements:

* AppBar with back arrow.
* Back arrow returns to previous screen.
* Include example settings:

  * Video Quality (Low/Medium/High)
  * Enable Notifications (Switch)
  * Dark Mode (Switch)
* Persist settings using SharedPreferences.

5. Help Screen
   Create HelpActivity.

Requirements:

* AppBar with back arrow.
* Back arrow returns to previous screen.
* Include help content explaining:

  * How to start a call
  * How to join a call
  * Permissions required
  * Troubleshooting tips

6. About Screen
   Create AboutActivity.

Requirements:

* AppBar with back arrow.
* Back arrow returns to previous screen.
* Display:

  * App name
  * Version
  * Description
  * Developer placeholder information

7. UI Requirements

* Material Design theme.
* Responsive layouts.
* Proper spacing and margins.
* Toolbar on all screens.
* Navigation drawer only on main screen.
* Back navigation follows Android standards.

8. Architecture
   Use:

* Activities + Fragments where appropriate.
* Separate packages:

  * activities
  * fragments
  * adapters
  * models
  * utils
  * webrtc
* Follow clean code practices.
* Add comments to important code sections.

9. Permissions
   Configure AndroidManifest.xml with:

* INTERNET
* CAMERA
* RECORD_AUDIO
* ACCESS_NETWORK_STATE

Implement runtime permission requests.

10. WebRTC
    Implement a basic working WebRTC integration.

Requirements:

* Local video stream.
* Remote video stream.
* Peer connection setup.
* Signaling service abstraction layer.
* Placeholder signaling implementation with clear TODO comments indicating where a production signaling server should be connected.

11. GitHub Actions
    Create a complete GitHub Actions workflow.

Requirements:

* Build on push and pull request.
* Use latest supported Java version for Android builds.
* Run:
  ./gradlew assembleDebug
* Upload APK artifact.

Create:
.github/workflows/android-build.yml

12. Deliverables
    Generate the complete project source code including:

* All Java files
* XML layouts
* Manifest
* Gradle files
* GitHub Actions workflow
* README.md

README must include:

* Project overview
* Build instructions
* Running instructions
* WebRTC notes
* GitHub Actions usage

13. Output Format
    Provide the full project tree first.

Then provide every file with:

* Full path
* Complete file contents

Do not omit any files.
Do not use placeholders such as "existing code here".
Generate a complete, buildable Android Studio project.

This prompt is detailed enough that a capable coding agent should generate the entire Android project structure, Gradle configuration, GitHub Actions workflow, navigation drawer, activities, and WebRTC scaffolding in one pass.
