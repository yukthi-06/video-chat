# User Request: stack trace to log file on mobile for crash analysis

## Goal
To write any uncaught exceptions occurring on any thread (including WebRTC native/negotiation/decoding background threads, EGL threads, and main UI threads) directly to a text log file on the mobile device, allowing easy retrieval and analysis by the user/AI.

## Solution Details
1. **Uncaught Exception Handler Utility**:
   - Created [CrashLogger.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/CrashLogger.java) to set a custom global default handler (`Thread.setDefaultUncaughtExceptionHandler`).
   - The logger intercepts crashes, generates a text file prefixed with `crash_yyyyMMdd_HHmmss.txt` inside `/sdcard/Vypeensoft/Video_Caller/logs/` (or falls back to the app sandbox logs folder if not writable), and dumps the device metadata (SDK version, model, manufacturer) and full stack trace.
   - It delegates execution back to the standard uncaught handler so that Android OS crash reporting behaves normally.
2. **App Startup Initialization**:
   - Updated [MainActivity.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/activities/MainActivity.java) to initialize `CrashLogger.init(getApplicationContext())` immediately in `onCreate()`.
 This captures and preserves all crashes to disk immediately on app launch.
