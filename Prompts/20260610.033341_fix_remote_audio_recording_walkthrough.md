# Walkthrough - Remote Audio Track Recording Support

We have resolved the remote audio recording limitation by tapping into the remote `AudioTrack` via the WebRTC `AudioSink` JNI interface, routing the received raw PCM audio samples to the remote recorder.

## Changes Made

### 1. WebRTC Client Component
#### [WebRtcClient.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/WebRtcClient.java)
- Extended the `WebRtcListener` interface with a callback `onRemoteAudioSample(byte[] data, int sampleRate, int channelCount)` to report incoming remote audio frames.
- Updated `onAddTrack()`: When a remote `AudioTrack` is received, we instantiate an anonymous `AudioSink` and register it on the track via `addSink()`. The `onData()` callback receives raw PCM frames and propagates them to the listener.

### 2. Main Call Screen Component
#### [CallActivity.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/activities/CallActivity.java)
- Enabled audio recording for the remote session by changing `remoteRecorder` initialization to set `recordAudio = true` (i.e. `new WebRtcVideoRecorder(remotePath, eglBase.getEglBaseContext(), true)`).
- Implemented the `onRemoteAudioSample()` listener callback to feed incoming remote audio buffers directly to `remoteRecorder.onAudioData(data, sampleRate, channelCount)`.
