# User Request: app still crashes on "Start a call"

## Problem
Even after addressing EGL issues, two other critical points in the initialization flow of `CallActivity` were susceptible to crashing:
1. **`getExternalFilesDir(null)` Nullability**: In `getRecordingsDirectory()`, if external storage is not mounted or has compatibility issues, `getExternalFilesDir(null)` can return `null`. The subsequent statement `new File(null, "...")` throws a `NullPointerException` which was outside the previous try-catch block, crashing the app.
2. **Unsupported Audio Device hardware features**: In `WebRtcClient.java`, the call to `JavaAudioDeviceModule.builder(context).setUseHardwareAcousticEchoCanceler(true).setUseHardwareNoiseSuppressor(true).createAudioDeviceModule()` will throw a `RuntimeException` or `NullPointerException` on devices/emulators whose audio hardware drivers do not support hardware echo cancelation or noise suppression.

## Solution Details
1. **Bulletproof Storage Path Resolution**:
   Updated the directory resolution in [CallActivity.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/activities/CallActivity.java) to catch all throwables on each path check:
   - Path 1: Root public SD card directory `/sdcard/Vypeensoft/Video_Caller/recordings`.
   - Path 2: Sandbox external files directory `getExternalFilesDir(null)` (with a check to make sure it's not null).
   - Path 3: Internal app files directory `getFilesDir()` (guaranteed to be writable and non-null).
2. **Robust Audio Device Module Creation**:
   Updated the `initWebRtc` method in [WebRtcClient.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/WebRtcClient.java) to wrap `JavaAudioDeviceModule` creation in a try-catch block. If it fails with hardware-accelerated effects, it automatically falls back to a plain `JavaAudioDeviceModule` builder. If that fails, it falls back gracefully to null.
