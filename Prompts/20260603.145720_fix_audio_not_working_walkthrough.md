# Walkthrough - Call Audio Fix

I have completed the integration fixes to resolve the missing audio stream.

## Changes Made

1. **WebRTC Java Audio Device Module (ADM)**:
   - Configured `JavaAudioDeviceModule` inside `WebRtcClient.java` with built-in hardware acoustic echo cancellers and noise suppressors enabled.
   - Connected this module by calling `.setAudioDeviceModule(audioDeviceModule)` on the `PeerConnectionFactory.Builder` object, ensuring WebRTC configures and binds to hardware microphones/speakers correctly.

2. **System Audio Manager Routing**:
   - Modified `CallActivity.java` to fetch the Android `AudioManager` service and set the mode to `MODE_IN_COMMUNICATION` during activity start, forcing sound to the speakerphone instead of the tiny earpiece.
   - Restored the mode to `MODE_NORMAL` and disabled the speakerphone inside `onDestroy()`.
