# Walkthrough: Split Call Recording Every 60 Seconds

We have updated the video recording system to automatically split video call recordings (for both local self-preview and remote stream) into 60-second chunks, outputting them as sequential MP4 files (e.g., `<timestamp>_local_1.mp4`, `<timestamp>_local_2.mp4`, etc.).

## Changes Made

### WebRTC Video Recorder

#### [MODIFY] [WebRtcVideoRecorder.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/WebRtcVideoRecorder.java)
- Constructor: Parses incoming `filePath` into `baseFilePath` and `fileExtension` (retaining directory and base filename).
- `startInternal()`: Constructs the segment's file name using `baseFilePath + "_" + segmentIndex + fileExtension`.
- `drawAndEncodeFrame()`: Monitors frame presentation timestamps (PTS). When `ptsNs >= 60,000,000,000` (60 seconds):
  1. Swaps the buffers for the final frame of the segment.
  2. Safely restores the caller's EGL context.
  3. Calls `splitRecording()`.
- `splitRecording()`: Safely stops and joins the encoder and audio encoder threads, releases all encoding resources via `releaseCodec()`, increments `segmentIndex`, and leaves `isStartRequested = true` so the next frame dynamically triggers `startInternal()` on the new segment filename with a reset PTS starting at 0.
- `stop()`: Log output updated to track the correct final file name.
