# User Request: capture above in an MD file

## Overview
This document compiles the comprehensive list of diagnostics and resolutions implemented for the Video Chat app call recording pipeline, permission requests, EGL thread contexts, and WebRTC signaling negotiation crashes.

---

## 1. Scoped Storage & Permission Restructure
- **Problem**: Writing files directly to `/sdcard/` on Android 10+ is blocked by Scoped Storage rules. `File.mkdirs()` returned `true` but subsequent `new MediaMuxer()` calls failed with `Permission denied`. Standard runtime permission checks also fail to grant root storage write access on Android 11+.
- **Resolution**:
  - Refactored `getRecordingsDirectory()` in [CallActivity.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/activities/CallActivity.java) to always perform a test-write (creating/deleting a file) to detect permissions. If the test fails, it falls back sequentially to `getExternalFilesDir(null)` (sandbox) and then to `getFilesDir()` (internal files).
  - Added `<uses-permission android:name="android.permission.MANAGE_EXTERNAL_STORAGE" />` in [AndroidManifest.xml](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/AndroidManifest.xml).
  - Implemented the settings intent redirect in [MainActivity.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/activities/MainActivity.java) to check and prompt for "All Files Access" on Android 11+ (API 30+) devices.

## 2. EglBase Initialization Mismatches
- **Problem**: Creating the recording context with `EglBase.CONFIG_RECORDABLE` on devices/emulators whose drivers do not support recordable surfaces caused the WebRTC engine to throw native `AssertionError`s, crashing the call screen immediately.
- **Resolution**:
  - Wrapped `EglBase.create()` inside [WebRtcVideoRecorder.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/WebRtcVideoRecorder.java) in a try-catch block.
  - Implemented a fallback to `EglBase.CONFIG_PLAIN` if recordable contexts are unsupported.
  - Broadened startup catch blocks to capture `Throwable` rather than just `IOException`.

---

## 3. Hardware Video Codecs Context Mismatch
- **Problem**: The `DefaultVideoEncoderFactory` and `DefaultVideoDecoderFactory` inside `WebRtcClient.java` were created with a `null` EGL context. Under hardware-accelerated modes, attempting to bind and publish textures without a shared context triggers immediate native crashes when start local capture is invoked.
- **Resolution**:
  - Propagated the shared `eglBase.getEglBaseContext()` from [CallActivity.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/activities/CallActivity.java) into the [WebRtcClient.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/WebRtcClient.java) constructor.
  - Passed this shared context directly into the encoder and decoder factory constructors.

---

## 4. Deferred Recording Activation
- **Problem**: Initializing and starting the H.264 encoders/muxers when the activity loaded resulted in zero-byte files, invalid files, or native setup failures because no video frames were flowing to the encoders.
- **Resolution**:
  - Removed immediate starts in `CallActivity.initWebRtcAndSignaling()`.
  - Deferred recording triggers until `onRemoteTrackAdded()` is executed, ensuring both local and remote feeds only start encoding after the connection handshake completes and remote video begins.
  - Isolated the starts in individual try-catch blocks to prevent connection drops.

---

## 5. Audio Device Module Permission Crash
- **Problem**: If `RECORD_AUDIO` permission was denied or not yet fully recognized by the module on app start, WebRTC's `JavaAudioDeviceModule.createAudioDeviceModule()` returned `null`, crashing the PeerConnection builder.
- **Resolution**:
  - Wrapped audio module building inside [WebRtcClient.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/WebRtcClient.java) in a robust try-catch structure, falling back to a plain builder and ignoring hardware AEC/NS effects if unsupported.
  - Added null safety checks to only register the module in `PeerConnectionFactory.Builder` if it was successfully initialized.

---

## 6. Remote ICE Candidate Race Condition
- **Problem**: When a guest joined, the creator immediately transmitted candidate packets. If network routing delivered candidates to the guest before they finished processing the remote SDP Offer, WebRTC's native C++ layers crashed on `addIceCandidate()`.
- **Resolution**:
  - Implemented `queuedRemoteCandidates` and `isRemoteDescriptionSet` inside [WebRtcClient.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/WebRtcClient.java).
  - Out-of-order candidates are queued if `isRemoteDescriptionSet` is false.
  - Intercepted `SdpObserver.onSetSuccess()` inside both `handleOffer()` and `handleAnswer()` to toggle the flag and flush all buffered candidates.

---

## 7. EGL Context Unbinding on Capture Thread
- **Problem**: Camera frames are processed on `CaptureThread`. When `onFrame()` is triggered, calling `recorderEglBase.makeCurrent()` bound the recorder's EGL context to `CaptureThread`, unbinding the camera's context. Subsequent frame updates threw `IllegalStateException: Unable to update texture contents`.
- **Resolution**:
  - Utilized `android.opengl.EGL14` inside `onFrame()` in [WebRtcVideoRecorder.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/WebRtcVideoRecorder.java) to fetch the active thread context, display, and read/write surfaces.
  - Placed a `finally` block to restore this original EGL state back onto the calling thread after writing the frame to the encoder, preserving `SurfaceTextureHelper` state.

---

## 8. On-Device Crash Logger
- **Problem**: Stack traces from background threads or native assertions are difficult to fetch without system logs.
- **Resolution**:
  - Created [CrashLogger.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/CrashLogger.java) to bind a custom global handler.
  - Dumps hardware specs, metadata, and stack traces to `crash_yyyyMMdd_HHmmss.txt` in `/sdcard/Vypeensoft/Video_Caller/logs/`.
