# Implementation Plan - Save Video Streams to Disk

This plan outlines how we will implement real-time recording of local and remote video streams directly to MP4 files on the device's storage. We will implement a GPU-accelerated encoder helper using Android's `MediaCodec`, `MediaMuxer`, and WebRTC's `EglRenderer`.

## User Review Required

> [!WARNING]
> - **Storage Permissions**: Writing to `/sdcard/...` requires `WRITE_EXTERNAL_STORAGE` and `READ_EXTERNAL_STORAGE` permissions. We will add these to `AndroidManifest.xml` and request them at app startup.
> - **Legacy Storage**: For Android 10 compatibility, we will enable `android:requestLegacyExternalStorage="true"` in the Manifest.
> - **Encoding Overhead**: Manually encoding H.264 streams on the GPU consumes battery and resources. We will run the encoding and muxing on a dedicated background thread to prevent UI and call stutter.

### WebRTC Recording Engine

#### [NEW] [WebRtcVideoRecorder.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/WebRtcVideoRecorder.java)
- A helper class implementing WebRTC's `VideoSink` interface.
- Configures and starts a `MediaCodec` video encoder (H.264, 640x480 resolution, 30 FPS).
- Obtains an input surface from `MediaCodec` and binds it to a private `EglRenderer` instance.
- Directs frames captured from `onFrame(VideoFrame)` to the OpenGL surface.
- Runs a background thread that polls output buffers from `MediaCodec` and writes them into `MediaMuxer` to assemble the `.mp4` file.

#### [MODIFY] [CallActivity.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/activities/CallActivity.java)
- In `initWebRtcAndSignaling()`, instantiate two `WebRtcVideoRecorder` objects (one for local video, one for remote video).
- File names will format as `/sdcard/Vypeensoft/Video_Caller/recordings/<TIMESTAMP>_local.mp4` and `/sdcard/Vypeensoft/Video_Caller/recordings/<TIMESTAMP>_remote.mp4`.
- Attach the recorders as sinks:
  - Add the local recorder as a sink to the local video track.
  - Add the remote recorder as a sink to the remote video track inside `onRemoteTrackAdded(VideoTrack)`.
- In `onDestroy()` or `endCall()`, stop both recorders to flush files cleanly.
