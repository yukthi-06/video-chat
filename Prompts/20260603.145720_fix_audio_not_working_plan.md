# Implementation Plan - Fix Missing Audio in Video Chat

This plan details fixing the missing audio stream during video chat. There are two causes:
1. **Audio Device Module (ADM)**: WebRTC on Android requires an explicit `AudioDeviceModule` (such as `JavaAudioDeviceModule`) to initialize the audio record/playout engine correctly.
2. **Audio Routing**: By default, Android often routes communication audio to the earpiece (very low volume) rather than the speakerphone. We will configure `AudioManager` to set communication mode and force output to the speakerphone.

## Proposed Changes

### Android Application Code

#### [MODIFY] [WebRtcClient.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/WebRtcClient.java)
- In `initWebRtc()`, instantiate `JavaAudioDeviceModule` using `JavaAudioDeviceModule.builder(context)`.
- Configure the builder to use hardware echo canceller and noise suppressor.
- Pass the created `AudioDeviceModule` into `PeerConnectionFactory.builder().setAudioDeviceModule(audioDeviceModule)`.

#### [MODIFY] [CallActivity.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/activities/CallActivity.java)
- In `onCreate()`, initialize the system `AudioManager`.
- Set the audio mode to `MODE_IN_COMMUNICATION` and enable the speakerphone.
- In `onDestroy()`, reset the audio mode to `MODE_NORMAL` and turn off the speakerphone.
