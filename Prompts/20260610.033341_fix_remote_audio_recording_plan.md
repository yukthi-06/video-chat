# Implementation Plan - Remote Audio Track Recording Support

We need to enable recording of the Remote Audio Track so that the remote peer's audio is captured in the remote recorder (`remoteRecorder`), making the remote `.mp4` file contain both remote video and remote audio.

## User Review Required

> [!IMPORTANT]
> Because WebRTC Java API structures vary between different builds (standard Google WebRTC vs. Jitsi vs. Conversations forks), we will attempt the integration using the standard WebRTC playout/sink callbacks available in modern libraries (like `im.conversations.webrtc:119.0.0`).
> We will need to run a Gradle compile check to verify which JNI interface hook is exposed (either `AudioTrack.addSink` or `JavaAudioDeviceModule` playout callback). We will request your permission to run compilation steps during the implementation.
