# Implementation Plan - WebSocket Signaling for VideoChatApp

This plan details how we will replace the local mock signaling client with a real, fully operational WebSocket signaling client. It uses a public, zero-configuration WebSocket pub/sub channel (`connect.ittysockets.com`) and OkHttp to broker real-time SDP and ICE candidate exchanges between two devices.

## User Review Required

> [!NOTE]
> - **Signaling URL**: We will use `wss://connect.ittysockets.com/videochatapp-{roomId}`. This is a public, zero-config broadcast channel.
> - **Self-Filtering**: Because the channel broadcasts messages to *all* subscribers in the channel (including the sender), we will attach a unique `senderId` to all payloads so each client ignores its own messages.
> - **Dependency Additions**: We will add the OkHttp library (`com.squareup.okhttp3:okhttp:4.12.0`) to the app's `build.gradle` to handle WebSockets.
