# Implementation Plan - Reflection-based Remote Audio Recording Support

Since the standard WebRTC Android SDK JNI does not expose track-level playout callbacks or `addSink` methods on `AudioTrack` in Java, we will implement a reflection-based interception mechanism (similar to how `flutter-webrtc` handles remote audio recording). 

By placing a utility class in the package `org.webrtc.audio` (where the internal classes are package-private), we can bypass access restrictions. We will swap WebRTC's internal `android.media.AudioTrack` instance with a wrapper `AudioTrackInterceptor` that captures raw PCM samples during playback and routes them to the remote recording session.

## User Review Required

> [!WARNING]
> This solution relies on Java Reflection to access private/package-private fields (`audioOutput` in `JavaAudioDeviceModule` and `audioTrack` in `WebRtcAudioTrack`). While this is the industry-standard workaround for mobile WebRTC audio recording, it is a hack that depends on internal SDK field names and structures.

## Proposed Changes

### 1. Reflection and Interception Components

#### [NEW] [AudioTrackInterceptor.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/AudioTrackInterceptor.java)
- Subclass `android.media.AudioTrack`.
- Wrap the original WebRTC `AudioTrack` and delegate all playback lifecycle calls (`play()`, `stop()`, `release()`, etc.) to it.
- Override `write(ByteBuffer, ...)` and `write(byte[], ...)` methods to copy the playout PCM data and forward it to a callback (`AudioDebugCallback`).

#### [NEW] [WebRtcAudioTrackUtils.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/org/webrtc/audio/WebRtcAudioTrackUtils.java)
- Place in package `org.webrtc.audio` to access package-private `WebRtcAudioTrack`.
- Implement `attachOutputCallback` and `detachOutputCallback` to swap the internal `AudioTrack` with `AudioTrackInterceptor` via Java Reflection.
