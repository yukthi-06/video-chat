# Implementation Plan - WebRTC Connection & Rendering Fixes

This plan outlines fixes to address two critical WebRTC issues causing empty video displays and no audio:
1. **Signaling Race Condition**: Currently, the Host generates the WebRTC Offer immediately upon connecting. If the Guest is not yet in the room, this Offer is lost. We will update the server and client to trigger the Offer only after the Guest joins.
2. **Unified Plan Track Callback**: Since our WebRTC configuration specifies `Unified Plan`, the remote streams are delivered via the `onAddTrack` callback, not `onAddStream`. We will implement `onAddTrack` and route the remote track to our renderer.
