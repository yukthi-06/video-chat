# Implementation Plan - Direct Peer Disconnection Detection

This plan details how to detect peer disconnections directly over the established WebRTC link without relying on the signaling server. We will monitor the native WebRTC `IceConnectionState`. If one peer closes their connection (or loses internet, or shuts down), the ICE state transitions to `DISCONNECTED` or `FAILED` on the other side, allowing the app to terminate the call session immediately.
