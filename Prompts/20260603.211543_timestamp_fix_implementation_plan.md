# Implementation Plan - Fix MP4 Duration and Timestamp Sync

Normalize both video and audio presentation timestamps (PTS) to start from 0 relative to the beginning of the recording, resolving the issue where short recordings show a duration of 2+ hours.

## User Review Required

> [!NOTE]
> We will offset video presentation timestamps by subtracting the first frame's timestamp in nanoseconds, matching the audio track which naturally starts at 0.

### WebRTC Call and Recording Component
