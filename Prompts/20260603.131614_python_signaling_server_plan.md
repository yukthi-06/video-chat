# Implementation Plan - Python Signaling Server

This plan details the addition of a Python-based WebSocket signaling server to the workspace. It will allow you to host your own local signaling server on your computer and connect your Android devices to it for WebRTC testing.

## User Review Required

> [!NOTE]
> - **Technology Stack**: Python 3.x using the `websockets` and `asyncio` libraries.
> - **Room Concept**: The server will support room routing. Clients will send a `"join"` message with a `roomId` on connection, and any message (offer, answer, ice candidates) sent to a room will be forwarded to all other peers in that room.
> - **Workspace Folder**: A new folder named `signaling_server` will be created in the project root containing the code.

### Android Code Alignment

> [!IMPORTANT]
> To connect your Android app to this Python server:
> 1. In `SignalingClient.java`, you will need to replace the `WS_BASE_URL` with:
>    `wss://YOUR_LOCAL_IP:8765` or `ws://YOUR_LOCAL_IP:8765` (depending on whether you set up SSL/WSS).
> 2. Ensure both your computer (running the python server) and your Android devices are on the same Wi-Fi network.
