# Implementation Plan - Fix Blank Self Preview

This plan details fixing the blank local video preview. The root cause is that the `SurfaceTextureHelper` in `WebRtcClient` was initialized with a `null` EGL context. In WebRTC, the camera capturer must be provided with a shared EGL context to successfully copy and display captured camera textures onto the `SurfaceViewRenderer`.

## Proposed Changes

### Android Application Code

#### [MODIFY] [WebRtcClient.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/WebRtcClient.java)
- Update `startLocalVideoCapture` method signature to accept `EglBase.Context eglContext`.
- Replace `SurfaceTextureHelper.create("CaptureThread", null)` with `SurfaceTextureHelper.create("CaptureThread", eglContext)`.

#### [MODIFY] [CallActivity.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/activities/CallActivity.java)
- In `initWebRtcAndSignaling()`, update the call to `startLocalVideoCapture` to pass `eglBase.getEglBaseContext()`.
