# Walkthrough - Audio Recording Support

Added support for recording the local participant's microphone audio during calls and saving it as an AAC audio track combined with the video in the local `.mp4` file.

## Changes Made

### WebRTC Call and Recording Component

#### [WebRtcVideoRecorder.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/WebRtcVideoRecorder.java)
- Extended the class with support for recording AAC audio (`recordAudio` flag).
- Integrated a dynamic AAC `MediaCodec` audio encoder and an asynchronous queue with a dedicated `AudioEncoderThread`.
- Synchronized `MediaMuxer` startup using a `muxerLock` to ensure both video and audio tracks are added before starting the muxer (when `recordAudio` is true).

#### [WebRtcClient.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/WebRtcClient.java)
- Exposed raw microphone callbacks using `JavaAudioDeviceModule.SamplesReadyCallback` and piped them through the updated `WebRtcListener` interface.

#### [CallActivity.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/activities/CallActivity.java)
- Enabled `recordAudio` for `localRecorder` and disabled it for `remoteRecorder` (due to platform/SDK playout capture limitations).
- Implemented `onLocalAudioSample` to pipe mic samples directly to the `localRecorder`.

## Verification Results

The build is ready to be compiled. You can build via Android Studio to run a manual verification on your test devices. The local `.mp4` files will now contain a synchronized audio stream alongside the video.
