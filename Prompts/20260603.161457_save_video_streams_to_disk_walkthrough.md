# Walkthrough - Save Video Streams to Disk

I have completed the features to record active video streams to `.mp4` container files on disk in real-time.

## Changes Made

1. **Storage Permissions & Legacy Access**:
   - Added `READ_EXTERNAL_STORAGE` and `WRITE_EXTERNAL_STORAGE` permissions to `AndroidManifest.xml`.
   - Enabled `android:requestLegacyExternalStorage="true"` to allow compatibility with devices targetting Android 10/Q.
   - Updated `MainActivity.java` to prompt the user to approve storage read/write permissions at startup alongside Camera/Mic permissions.

2. **GPU Video Encoder & Muxer (`WebRtcVideoRecorder.java`)**:
   - Implemented `WebRtcVideoRecorder` which implements `VideoSink`.
   - Configures an Android `MediaCodec` instance utilizing standard AVC/H.264 profile properties (640x480 resolution, 30 FPS, 1.5 Mbps bitrate).
   - Links the encoder surface to a private `EglRenderer` instance that clear/swaps frames, feeding captured raw `VideoFrame` inputs directly into the encoder on the GPU.
   - Spins up a background thread (`VideoEncoderThread`) to dequeue encoded frame outputs and append them into `MediaMuxer` chunks.

3. **Call Binding**:
   - Updated `CallActivity.java` to format dynamic file paths under `/sdcard/Vypeensoft/Video_Caller/recordings/` containing the session startup timestamp.
   - Instantiated and started two recorders (`localRecorder` and `remoteRecorder`) using the shared rendering context.
   - Registered them as track sinks under `onLocalStreamReady()` and `onRemoteTrackAdded()`.
   - Guaranteed clean shutdown and resource releasing in both `endCall()` and `onDestroy()`.
