# User Request: file read/write permissions to be given

## Problem
In Android 11+ (API level 30+), standard `WRITE_EXTERNAL_STORAGE` and `READ_EXTERNAL_STORAGE` permissions are deprecated and restricted by Scoped Storage. This prevents apps from creating or writing files in custom public folders like `/sdcard/Vypeensoft/Video_Caller/recordings/` unless the application requests and the user grants "All Files Access" (`MANAGE_EXTERNAL_STORAGE`).

Without this permission, the app is forced to fallback to the private application sandbox folder, which is hidden deep in the system storage and cannot be easily viewed by standard user file managers.

## Solution Details
1. **Declare Permission**: Added `<uses-permission android:name="android.permission.MANAGE_EXTERNAL_STORAGE" tools:ignore="ScopedStorage" />` in [AndroidManifest.xml](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/AndroidManifest.xml).
2. **Request Permission at Runtime**:
   Updated the runtime permission checks in [MainActivity.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/activities/MainActivity.java):
   - Exclude deprecated storage permissions from the default dialog on Android 11+.
   - Detect if the device is running Android 11+ (API 30+) and check if `Environment.isExternalStorageManager()` returns `false`.
   - If not granted, direct the user to the system Settings page (`Settings.ACTION_MANAGE_APP_ALL_FILES_ACCESS_PERMISSION` or `Settings.ACTION_MANAGE_ALL_FILES_ACCESS_PERMISSION`) to authorize "All Files Access" for the app.
