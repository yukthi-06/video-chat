# Implementation Plan - Scoped Storage Writable Directory Fallback

This plan details resolving the issue where the `recordings` directory might be created, but no MP4 files are generated. 

On Android 11 (API 30) and above, **Scoped Storage** blocks apps from writing directly to the root external storage (`/sdcard/`) even if storage permissions are granted. To guarantee files are created, we will implement a fallback mechanism: if writing to `/sdcard/` fails or is blocked, the app will automatically route files to its private external files directory (`/sdcard/Android/data/com.example.videochatapp/files/Vypeensoft/...`), which is always writable on all Android versions.

- Implement a helper method `getRecordingsDirectory()` to check if the root `/sdcard/Vypeensoft/...` directory is writable.
- If it is not writable (or `mkdirs()` fails), fallback to the directory returned by `getExternalFilesDir(null)`.
- Log the selected directory path and show a Toast so the user knows where the files are being saved.
