# Walkthrough - Fix MP4 Duration and Timestamp Sync

Fixed the issue where recorded MP4 files showed an incorrect playback duration (e.g. 2 hours or more for a short call).

## Changes Made

### WebRTC Call and Recording Component

#### [WebRtcVideoRecorder.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/WebRtcVideoRecorder.java)
- Introduced `firstVideoTimestampNs` to record the timestamp of the first frame.
- Normalized incoming video frames' presentation timestamps by subtracting the offset before passing them to the encoder's input surface via `recorderEglBase.swapBuffers(ptsNs)`.
- Reset `firstVideoTimestampNs` to `-1` when release codec occurs.

## Verification Results

This normalization ensures that both audio and video tracks start exactly at `0` relative to the beginning of the recording. This resolves duration computation errors in media players and seeks bars. The code is ready for compile and deployment.
