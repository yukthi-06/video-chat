# Split Call Recording Every 60 Seconds

This implementation plan outlines the changes required to split the video call recording files every 60 seconds. Each segment will be saved as an individual MP4 file named sequentially (e.g. `_1.mp4`, `_2.mp4`, etc.).

## Proposed Changes

### Video Recorder Component

#### [MODIFY] [WebRtcVideoRecorder.java](file:///x:/Projects_X/0_Active/8_Android_APK/04_Early_Stage/Video_Chat_APK_GITHUB/app/src/main/java/com/example/videochatapp/webrtc/WebRtcVideoRecorder.java)
- Split the initial `filePath` into `baseFilePath` and `fileExtension` in the constructor.
- Add a `segmentIndex` field starting at 1.
- In `startInternal()`, construct the current segment's file path as `baseFilePath + "_" + segmentIndex + fileExtension`.
- In `drawAndEncodeFrame()`, monitor `ptsNs`. When `ptsNs >= 60_000_000_000L` (60 seconds), trigger a segment split.
- Implement `splitRecording()` which cleanly stops/releases the current encoders/muxer, increments `segmentIndex`, and leaves `isStartRequested = true` so that the next video frame triggers `startInternal()` on the next segment.
